### QUESTÃO 1
Interessante observar que, em Java, a construção do código é mais organizada e legível do que nas demais linguagens. Python, de longe, é a mais fácil e apresenta a menor quantidade de linhas, juntamente com Swift, a simplicidade de ambas é o ponto-chave. E, por último, em Haskell eu não consegui entender nada, sem dúvidas, é a linguagem com a gramática mais estranha que já vi.

### QUESTÃO 2
Mesmo resolvendo o mesmo problema, cada linguagem teve seu próprio jeito de trabalhar. C passou uma sensação de maior controle, Ruby deixou tudo mais fácil parecendo python, e Erlang trouxe uma solução mais enxuta e diferente do comum. Para mim, C foi a mais simples de ler pela semelhança com Java.

### QUESTÃO 3
Python ficou mais simples de ler como sempre, Java manteve uma estrutura parecida com C, e Haskell usou uma solução diferenciada, pouco comum na minha opnião. Python foi a mais clara das outras.

### QUESTÃO 4
C ficou muito parecido com o código original, passando mais controle. Python deixou a lógica mais fácil de acompanhar. Kotlin, Rust e Swift trouxeram uma escrita mais moderna e organizada, cada uma adaptando o laço ao seu próprio estilo. Para mim, foi interessante perceber como a mesma ideia pode ser escrita de formas tão diferentes, e entre elas Python e Kotlin foram as que achei mais confortáveis de ler.
